Privacy Flip автоматически управляет параметрами приватности устройства в зависимости от состояния блокировки/разблокировки экрана. При блокировке устройство может отключать Wi-Fi, Bluetooth, мобильные данные, службы определения местоположения, NFC и даже датчики камеры/микрофона. При разблокировке приложение аккуратно восстанавливает те функции, которые вы хотите вернуть.

<b>Работает с root-доступом ИЛИ <a href="https://github.com/RikkaApps/Shizuku">Shizuku</a> — выберите предпочитаемый способ привилегий.</b>

<b>Функции</b>

• Управление приватностью устройства в зависимости от блокировки/разблокировки экрана.
• Двойная поддержка привилегий: root-доступ ИЛИ Shizuku (вариант, не требующий root-доступа).
• Автоматическое определение способа привилегий (<a href="https://github.com/RikkaApps/Sui">Sui</a> → root → Shizuku).
• Настраиваемое управление функциями (Wi-Fi, Bluetooth, мобильные данные, службы местоположения, NFC).
• Экспериментальные средства управления доступом к камере и микрофону (Android 12+).
• Пользовательские задержки в миллисекундах/секундах для блокировки и восстановления (0–60 секунд).
• Фоновый сервис для постоянного отслеживания и применения настроек.
• Интеграция с плиткой Быстрых настроек.
• Классический для Android вид аккуратного и эффективного интерфейса.
• Подробное журналирование.
• Инструменты отладки.

<b>Настраиваемые функции приватности</b>

• «Wi-Fi» — отключение/включение интерфейса IEEE 802.11.
• «Bluetooth» — отключение/включение Bluetooth.
• «Mobile data» — управление передачей мобильных данных.
• «Location services» — управление службами местоположения (при помощи GPS, Wi-Fi и сотам мобильной связи).
• «NFC» — управление модулем NFC.

<b>Экспериментальные функции (Android 12+)</b>

• «Camera» — отключение/включение доступа к камере.
• «Microphone» — отключение/включение доступа к микрофону.

Примечание: управление камерой и микрофоном требует окна времени между блокировкой экрана и блокировкой устройства. Приложение предупредит, если на устройстве настроена мгновенная блокировка.

<b>Требования</b>

• Android 7.0+ (API level 24).
• Android 12+ (API level 31) для функций камеры/микрофона.

<b>Выберите ОДИН способ привилегий</b>

<b>Вариант 1: root-доступ</b> (рекомендуется для рутированных устройств)
• root-доступ через <a href="https://github.com/topjohnwu/Magisk">Magisk</a>, <a href="https://supersuroot.org">SuperSU</a> или аналогичные решения;
• наилучшая производительность при привилегиях UID 0.

<b>Вариант 2: Shizuku</b> (root-доступ не требуется)
• установлено и запущено приложение Shizuku;
• привилегии предоставлены через ADB (USB) или беспроводную отладку;
• удобно для устройств без root-доступа.

<b>Вариант 3: Sui</b> («лучшее из обоих миров»)
• устройство с root-доступом и установленным модулем Sui для Magisk;
• автоматический запуск и отсутствие лишних запросов.

Приложению требуются повышенные привилегии для управления системными функциями приватности.

<b>Приватность и безопасность</b>

• Никакой телеметрии — данные не собираются, и никуда не передаются.
https://reports.exodus-privacy.eu.org/reports/io.github.dorumrr.privacyflip/latest
• Только локальное хранение — все настройки остаются на устройстве, все действия выполняются только локально.
• Отсутствие сетевых разрешений — приложение не может выходить в интернет.
• Открытый исходный код — полная прозрачность кода.

<b>Использование</b>

<b>Для пользователей с root-доступом:</b>
1. откройте PrivacyFlip и предоставьте root-права при запросе;
2. настройте желаемые параметры приватности;
3. при необходимости задайте задержки для блокировки/восстановления;
4. приложение будет автоматически работать в фоновом режиме.

<b>Для пользователей Shizuku:</b>
1. установите и запустите приложение Shizuku (включите через ADB или беспроводную отладку);
2. откройте PrivacyFlip и предоставьте разрешения Shizuku при запросе;
3. настройте желаемые параметры приватности;
4. при необходимости задайте задержки для блокировки/восстановления;
5. приложение будет автоматически работать в фоновом режиме.

После настройки PrivacyFlip работает автоматически:
• блокировка устройства → выбранные функции будут отключены;
• разблокировка устройства → выбранные функции будут восстановлены.
